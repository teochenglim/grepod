# v1.0.0 — Stabilize & commit to compatibility

**Status:** not started.

## Goal

Not a features release — a promise release. Everything in scope for 1.0
(tests, observability, live tail) landed in v0.2.0–v0.6.0; this version
is where grepod commits to not breaking the people using it.

## Planned scope

- **Compatibility commitment**, written down (likely a `COMPATIBILITY.md`
  or a section in `README.md`): SemVer from here on; documented env vars,
  `/api/search` and `/api/tail` query params/response shapes, and Helm
  `values.yaml` keys are considered stable and won't change without a
  major version bump.
- **What's explicitly still out of scope**, stated plainly rather than
  left implicit: built-in auth (stays "put a proxy in front," see
  [k8s/README.md#exposing-it-safely](../k8s/README.md#exposing-it-safely))
  and multi-namespace support (stays one release per namespace, see
  [DESIGN/01](../DESIGN/01_design_overview.md#non-goals)). Both were
  considered for 1.0 and deliberately deferred, not overlooked — worth
  re-litigating for 2.0 if real usage asks for them, not before.
- **Final security pass**: re-run Semgrep + Trivy fs/image scans
  (`security.yml` already runs these on every tag) and confirm zero
  CRITICAL/HIGH before tagging; review the RBAC `Role` one more time to
  confirm it's still exactly `get/watch/list` on `pods` + `get` on
  `pods/log` and nothing has crept in.
- Tag `v1.0.0` via `make release VERSION=1.0.0`.
