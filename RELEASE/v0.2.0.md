# v0.2.0 — Architecture-driven test suite

**Status:** shipped, tagged `v0.2.0`.

## Goal

CI has run `go test ./...` since v0.1.0 with nothing for it to test. Close
that gap — not by chasing a coverage percentage, but by asserting the
specific behaviors each `DESIGN/0X` doc already claims the code has. A
test that doesn't trace back to a documented behavior probably shouldn't
be written; a documented behavior without a test is a gap.

## Bugs found and fixed

Writing tests against the documented behavior — rather than against
whatever the code happened to already do — surfaced three real bugs, none
previously caught because nothing exercised these paths before:

- **Cross-shard search was completely broken.** `Store.Search`'s
  `snippet(%[1]s.fts, ...)`/`bm25(%[1]s.fts)`/`%[1]s.fts MATCH ?` passed a
  schema-qualified table name (`d20260712.fts`) into FTS5 auxiliary
  function arguments and the `MATCH` expression. FTS5 only resolves a
  schema-qualified name in the `FROM` clause — inside function arguments
  and `WHERE`, `schema.table` is parsed as `table.column` instead, so
  every search failed outright with `no such column`/`no such table`.
  Fixed by aliasing the attached table back to the unqualified `fts` in
  `FROM %[1]s.fts AS fts`, so `snippet(fts, ...)`, `bm25(fts)`, and
  `fts MATCH ?` all resolve correctly.
- **Any search term containing FTS5 query-syntax characters (starting
  with a hyphen, e.g. `flush-on-size-marker`) errored instead of
  searching.** The raw user query was passed straight into `MATCH ?`
  unescaped, so it was parsed as FTS5's own query language rather than as
  a literal search string — a real problem given pod names, UUIDs, and
  stack traces routinely contain hyphens and colons. Fixed with
  `sanitizeMatchQuery`: each whitespace-separated term of the input is
  now quoted as its own FTS5 phrase (implicitly ANDed together), so
  punctuation in the query is matched literally instead of parsed as an
  operator.
- **`BatchQueue.Close()` didn't actually wait for the final flush**, and
  could drop a line enqueued immediately before `Close()` entirely. `run()`'s
  `select` could pick the now-ready `<-q.done` case over a simultaneously
  ready `<-q.in` case (Go picks pseudo-randomly between ready cases),
  skipping that line's flush; and `Close()` returned as soon as
  `close(q.done)` ran, before `run()`'s goroutine had actually finished
  flushing — so callers (like `main.go`'s `defer queue.Close()` ahead of
  `defer store.Close()`) had no real guarantee the store wasn't closed
  out from under an in-flight flush. Fixed: `run()`'s shutdown path now
  drains `q.in` before its final flush, and `Close()` blocks on a new
  `stopped` channel that `run()` closes only after that flush completes.

## Delivered scope

- **`internal/tailer`** (against [DESIGN/02](../DESIGN/02_design_tailer.md)):
  - New container → tailer started; restart count change → old goroutine
    stopped and a fresh one started; no change → no-op (§ Reconciliation).
  - Previous-container-log fetch is best-effort and swallows "no previous
    instance" errors without failing the tailer (§ Streaming a container).
  - Exponential backoff caps at 5s and resets don't happen mid-stream.
  - Uses a fake `kubernetes.Interface` (client-go's `fake` package) and a
    fake `Sink`, not a real cluster.
- **`internal/storage`** (against
  [DESIGN/03](../DESIGN/03_design_storage.md)):
  - `BatchQueue` flushes at `size` lines or `interval`, whichever first;
    `Enqueue` drops rather than blocks when full.
  - `InsertBatch` splits a batch across the correct daily shards.
  - `Search` only attaches shards that exist in range, ranks by `bm25()`,
    and caps results at 500 regardless of the caller's `limit`.
  - `CleanupOldDBs` deletes shards older than `retentionDays` and leaves
    the rest.
  - Uses a temp dir per test (`t.TempDir()`), real SQLite (it's pure Go —
    no CGO toolchain needed in CI).
- **`internal/api`** (against
  [DESIGN/04](../DESIGN/04_design_api.md)):
  - `/api/search` param validation: missing `q` → 400, bad dates → 400,
    `end` before `start` → 400, defaults to today when `start`/`end`
    omitted.
  - `/static/*` and `/` serve the expected content types via
    `httptest.NewServer` (the same pattern used to smoke-test the chart
    during scaffolding).
- `ci.yml`'s `go test` step keeps `-race -cover`; coverage is reported but
  not gated — the gate is "does every documented behavior have an
  assertion," which a percentage can't tell you.
