# v0.1.0 — Core tail-index-search loop

**Status:** in progress, not yet tagged.

## Scope

- Namespace-scoped pod log tailing via a `client-go` informer
  (`internal/tailer`), including previous-container-log recovery on crash
  and exponential-backoff stream reconnection.
- Batched, non-blocking ingestion into daily-sharded SQLite FTS5 databases
  (`internal/storage`), with a nightly retention cron.
- Cross-shard full-text search over `/api/search`, ranked by `bm25()` with
  highlighted snippets (`internal/api`).
- Single dependency-free embedded search UI (`web/templates/index.html` +
  `web/static/{style.css,app.js,favicon.svg}`).
- Single static binary, single container image, single PVC — no external
  dependencies (no Loki, no message queue, no separate DB).
- Kubernetes RBAC scoped to `get/watch/list` on `pods` and `get` on
  `pods/log`, namespace-scoped `Role`/`RoleBinding` (not cluster-wide).
- Deployable via plain manifests (`k8s/`) or Helm (`helm/`).

## Known constraints

- Single replica only — see
  [DESIGN/03](../DESIGN/03_design_storage.md#why-not-horizontal-scale-out).
- One grepod instance watches exactly one namespace.

## Bugs found and fixed

- `//go:embed static/*` in `cmd/server/main.go` looked for
  `cmd/server/static/`, which didn't exist — the static UI files lived at
  the repo root. Fixed by moving them to `web/static/` behind a dedicated
  `web` package (`web/web.go`) that owns the embed, since `go:embed`
  patterns can't reach outside their declaring file's directory.
- `go.sum` was missing entirely, so `go build`/`go test`/`docker build`
  would all fail on a clean checkout. Regenerated via `go mod tidy`.
- `helm/templates/deployment.yaml` set `podManagementPolicy: Parallel`,
  which isn't a valid `Deployment` field (it's `StatefulSet`-only) and
  would be rejected under strict/server-side validation. Removed.
- `helm/templates/configmap.yaml` was missing `metadata.namespace`, unlike
  every other template in the chart — it would have landed in whatever
  namespace `helm install --namespace` used rather than the namespace
  grepod actually watches. Fixed to match the rest of the chart.
