# v0.6.0 — Hardening, perf pass, docs catch-up

**Status:** not started.

## Goal

Everything functional for 1.0 exists by the end of v0.5.0. This release
is about making sure it holds up and that the docs actually describe what
shipped — the last thing before committing to compatibility in
[v1.0.0](v1.0.0.md).

## Planned scope

- **Perf pass** on the two things most likely to degrade under a busy
  namespace: `BatchQueue` throughput (tune default `BATCH_SIZE`/
  `BATCH_INTERVAL` against a realistic line rate) and `Store.Search`
  latency across many attached shards. Document the numbers and the
  method (so they can be re-run, not just cited) in a new
  `DESIGN/05_design_performance.md`.
- **Optional e2e smoke test in CI** — `scripts/local-test.sh` already
  spins up a `kind` cluster and installs the Helm chart for manual use;
  evaluate wiring an equivalent (kind + install + hit `/healthz`,
  `/api/search`, `/api/tail`) into `ci.yml` as a non-blocking job, given
  the added CI runtime/complexity cost.
- **Docs catch-up**: re-read `README.md`, `ARCHITECTURE.md`, all of
  `DESIGN/`, `k8s/README.md`, `helm/README.md` end to end against what
  v0.2.0–v0.5.0 actually shipped (new endpoints, new env vars, new Helm
  values, new probes) — fix drift rather than let it accumulate into 1.0.
- Full `RELEASE/v0.1.0.md`-style "Bugs found and fixed" retrospective
  across v0.2.0–v0.6.0 collected here, so 1.0.0 starts from a clean slate.
