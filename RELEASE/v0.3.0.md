# v0.3.0 — Observability foundation: health, structured logging, log level

**Status:** shipped, tagged `v0.3.0`.

## Goal

Two things grepod needed before it's trustworthy to run unattended:
liveness/readiness signals separate from the UI route, and structured
logs for grepod's own operation. Alongside that, the ingested-log schema
gained a `level` column — the foundation [v0.5.0](v0.5.0.md)'s UI
filtering builds on. `/metrics` (RED metrics) is deliberately **not** in
this release — see [v0.7.0](v0.7.0.md), where it lands after the
tail/query surfaces it measures actually exist.

## Delivered scope

- **`/healthz`** — pure liveness, always `200`.
- **`/readyz`** — `200` once `tailer.Manager.Ready()` reports the pod
  informer's initial cache sync completed, `503` before that.
  `k8s/20-deployment.yaml` and `helm/templates/deployment.yaml`'s
  `readinessProbe`/`livenessProbe` now point at `/readyz`/`/healthz`
  instead of `/`.
- **Structured logging** — `log.Printf`/`log.Fatalf` across `cmd/server`,
  `internal/tailer`, `internal/storage` replaced with `log/slog`: JSON
  handler, `LOG_LEVEL` env var (default `info`), every line tagged with
  fixed `pod_namespace`/`pod_name` attributes.
- **`pod_namespace`/`pod_name`/`NAMESPACE` now come from the Kubernetes
  Downward API**, not manual config: sourced via `valueFrom.fieldRef`
  (`metadata.namespace`/`metadata.name`) directly on the container in
  both `k8s/20-deployment.yaml` and `helm/templates/deployment.yaml`.
  This *replaced* the old `NAMESPACE` key in
  `k8s/10-configmap.yaml`/`helm/templates/configmap.yaml` — grepod always
  watches its own namespace, so it can no longer drift from where the
  chart/manifests were actually deployed (previously a manually-kept-
  in-sync value, called out as a footgun in the READMEs before this).
- **`level` column** on the ingested log schema: `storage.LogLine` and
  the `fts` table gained a `level` field alongside
  `pod`/`namespace`/`container`/`timestamp`/`line`. Populated by
  `tailer.detectLevel` — a single regex matching a whole-word level token
  (`FATAL`/`ERROR`/`WARN`/`WARNING`→`WARN`/`INFO`/`DEBUG`/`TRACE`,
  case-insensitive), empty when nothing matches. `storage.SearchResult`
  and the `/api/search` JSON response expose it. UI consumption (tabs,
  severity-and-above filtering, grouping) is deferred to
  [v0.5.0](v0.5.0.md) — this release only adds and surfaces the data.
- **`/api/search` defaults to the past 7 days, not just today**, when
  `start` is omitted (inclusive — today plus 6 days back). Configurable
  via `DEFAULT_SEARCH_DAYS` (new env var, default `7`), threaded through
  as a `Handler` field rather than hardcoded. `web/static/app.js`'s date
  pickers default to the same window.
- **Kustomize support for the plain manifests**: `k8s/kustomization.yaml`
  lists the core resources and centralizes the deploy-target namespace —
  `kubectl apply -k k8s/`, or `kustomize edit set namespace <ns>` to
  target a different one, instead of hand-editing `metadata.namespace`
  across every file. Pairs directly with the Downward API change above:
  the namespace grepod watches and the namespace it's deployed into are
  now definitionally the same value, sourced from the pod's own runtime
  metadata rather than two independently-set config values that could
  drift apart.
- Confirmed already in place, not new work: `.github/workflows/release.yml`'s
  `docker` job already builds and pushes to `ghcr.io/teochenglim/grepod`
  on every tagged release — no changes needed here.

## Breaking, pre-1.0

- The `fts` schema change means existing local `/data/logs_*.db` shards
  from before this release don't have the `level` column. No migration —
  delete and re-ingest, same as any other pre-1.0 schema change.
- `k8s/10-configmap.yaml`/`helm/templates/configmap.yaml` no longer set
  `NAMESPACE`. Anyone who forked/customized these manifests and relied on
  overriding `NAMESPACE` via the ConfigMap needs to remove that override —
  it's now always sourced from the pod's own `metadata.namespace`.

## Verified

`go build`/`go vet`/`gofmt`/`go test -race -cover` (30/30), `helm lint` +
full `helm template` render (ingress + autoscaling enabled), `kubectl
kustomize k8s/` (default namespace and after `kustomize edit set
namespace`), `docker build`, and an `httptest` smoke run exercising
`/healthz`, `/readyz` (both states), and `/api/search` end-to-end
including the `level` field round-tripping through ingestion → storage →
JSON response.
