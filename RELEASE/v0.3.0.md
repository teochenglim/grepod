# v0.3.0 — Observability

**Status:** not started.

## Goal

Today `/` doubles as both the UI and the only thing `readinessProbe`/
`livenessProbe` can point at, there's no metrics surface, and logging is
unstructured `log.Printf`. Give grepod the minimum an operator needs to
run it unattended: separate health signals, a way to see ingestion/search
behavior over time, and logs that a log processor (ironically, maybe
grepod itself) can parse.

## Planned scope

- **`/healthz`** — pure liveness: process is up, HTTP server is serving.
  No dependency checks.
- **`/readyz`** — readiness: `storage.Store` opened successfully and the
  `tailer.Manager`'s informer cache has synced at least once. Fails fast
  during startup instead of accepting traffic before the tailer is
  actually watching pods.
- **`/metrics`** — Prometheus text format, no external dependency beyond
  `client_golang`. At minimum:
  - `grepod_lines_ingested_total` / `grepod_lines_dropped_total` (counter,
    the latter for `BatchQueue`'s full-channel drop path)
  - `grepod_batch_flush_duration_seconds` (histogram)
  - `grepod_search_duration_seconds` / `grepod_search_results_total`
  - `grepod_tailed_containers` (gauge, current count)
- **Structured logging** — replace `log.Printf` call sites with `log/slog`
  (stdlib, no new dependency), JSON handler by default, `LOG_LEVEL` env
  var (default `info`).
- Update `k8s/20-deployment.yaml` and `helm/templates/deployment.yaml`
  probes to point at `/healthz`/`/readyz` instead of `/`; add a `metrics`
  port + a `prometheus.io/scrape` annotation (or a `ServiceMonitor`
  template behind a values flag, if the cluster has the Prometheus
  Operator) to both.
- Document all three endpoints in [DESIGN/04](../DESIGN/04_design_api.md)
  and the top-level [README.md](../README.md) configuration table.
