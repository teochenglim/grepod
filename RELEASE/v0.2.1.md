## Fixes

- **Semgrep** (`yaml.kubernetes.security.allow-privilege-escalation-no-securitycontext`):
  the `grepod` container in `k8s/20-deployment.yaml` had no container-level
  `securityContext`, so `allowPrivilegeEscalation` defaulted to unset
  (effectively `true`). Added `securityContext: {allowPrivilegeEscalation:
  false, capabilities: {drop: ["ALL"]}}` to both `k8s/20-deployment.yaml`
  and `helm/templates/deployment.yaml` (the Helm chart isn't valid YAML on
  its own â€” Semgrep's YAML scanner skips `.tpl`-style Go template syntax â€”
  so it wasn't flagged there, but the same gap existed and got the same
  fix).
- **Trivy filesystem** (6 HIGH CVEs in `go.mod`): `golang.org/x/net`
  v0.26.0 (5 CVEs, incl. CVE-2026-25681/27136 XSS-via-HTML-parsing and
  CVE-2026-33814 HTTP/2 DoS) and `golang.org/x/oauth2` v0.21.0 (1 CVE),
  both transitive via `k8s.io/client-go`. Bumped via
  `go get golang.org/x/net@latest golang.org/x/oauth2@latest` + `go mod
  tidy` â†’ `x/net` v0.57.0, `x/oauth2` v0.36.0. This also bumped `go.mod`'s
  `go` directive to `1.25.0` (pulled in by the newer `x/net`).
- **Trivy image** (10 HIGH Go-stdlib CVEs, e.g. CVE-2026-32280/32281/32283
  crypto/tls/x509 DoS, CVE-2026-39822 os.Root symlink traversal): the
  `golang:1.23-alpine` build-stage image embedded an unpatched Go
  toolchain/stdlib into the compiled binary â€” Trivy's `gobinary` scanner
  detects this from the binary's embedded build info even though the
  final image is distroless. Bumped `Dockerfile`'s build stage to
  `golang:1.26-alpine`. `.github/workflows/{ci,release}.yml`'s
  `GO_VERSION` bumped from `1.23` to `1.26` to match â€” CI now builds/tests
  with the same major.minor the release image ships.
